# Email ↔ Task/Idea Quick Capture — Implementation Plan

## The Problem

Multipassionate creatives and solopreneurs read hundreds of emails daily. Right now, IdeaBox's AI already *extracts* actions and ideas from emails, but the path from "I see this in an email" to "it's saved as a task/idea on my board" has too much friction:

1. **Actions are display-only in email detail** — You see "Reply to Sarah about contract" in the analysis card, but can't promote it to a project_item without navigating to Tasks > Triage
2. **Idea save is nearly invisible** — A tiny bookmark icon on hover, no project selection, no confirmation feedback
3. **Summary items can't be captured** — The daily summary says "Invoice from Acme is overdue" but there's no way to save that as a task
4. **Key points are read-only** — ContentDigest shows useful bullet points but no save affordance
5. **No Task → Email flow** — Can't quickly fire off an email from a task context

## Design Principles (for multipassionate creatives)

1. **Capture in context** — Never force a page change to save something
2. **Smart defaults** — AI already analyzed this; pre-fill everything, let me confirm with one click
3. **Type inference** — AI knows if this is a task vs idea; surface that
4. **Project inference** — If the email is from client X, suggest their project
5. **Minimal friction** — One click to accept, two clicks max to customize
6. **Bidirectional links** — Every saved item links back to source email; tasks can reach their email

## Feature A: Smart Capture Bar on Email Detail

**New component: `SmartCaptureBar.tsx`**

A visually distinct section at the bottom of the AI Analysis card in EmailDetail that surfaces all capturable items from the AI analysis, pre-filled and ready to save.

```
┌─────────────────────────────────────────────────────────────┐
│ ⚡ Quick Capture                                            │
│                                                             │
│  📋 Reply to Sarah about contract terms          [Save →]  │
│  📋 Schedule meeting for March 15                [Save →]  │
│  💡 Use their pricing model for our service      [Save →]  │
│                                                             │
│                    [+ Add your own task or idea]            │
└─────────────────────────────────────────────────────────────┘
```

### Behavior
- **Auto-populated** from email's AI analysis (actions from `actionExtraction` + ideas from `ideaSparks`)
- Each row shows: type icon (task/idea), pre-filled title, and a **"Save"** button
- Clicking "Save" opens the existing **QuickAcceptPopover** (project dropdown + priority selector) — 2 clicks total
- Project dropdown **smart-defaults** to: the project linked to the email's contact/client (if matched), or MRU project
- Clicking the title text makes it **inline-editable** so users can tweak before saving
- **"+ Add your own"** button opens the existing `CreateItemDialog` pre-filled with email context (subject, sender, email_id)
- Items that have already been saved (check `email_ideas` and `actions` status) show a subtle "Saved" checkmark instead
- Appears even when there are zero AI suggestions — just shows the "+ Add your own" button, because sometimes the user wants to capture something AI didn't catch

### Files
- **NEW**: `src/components/email/SmartCaptureBar.tsx` (~200 lines)
- **EDIT**: `src/components/email/EmailDetail.tsx` — render SmartCaptureBar after the analysis card sections, pass email + analysis data as props

### Data flow
```
SmartCaptureBar receives: email, analysis (from useEmailAnalysis)
  → builds capturable items list from analysis.actionExtraction.actions + analysis.ideaSparks.ideas
  → user clicks Save → QuickAcceptPopover → calls useProjectItems.createItem()
  → creates project_item with source_email_id = email.id, source_action_id (if from action)
  → also calls POST /api/ideas (if idea type) to persist in email_ideas
```

## Feature B: Inline Save from Content Digest Key Points

**Enhancement to `ContentDigestSection.tsx`**

Each key_point bullet already renders in the content digest. Add a small "+" button (matching the existing golden nugget save pattern) that saves the key point as an idea.

### Behavior
- On hover, a small "+" icon appears to the right of each key point
- Clicking it calls `POST /api/ideas` with:
  - `idea`: the key point text
  - `ideaType`: inferred from email category (newsletter → `learning`, client → `business`, etc.)
  - `emailId`: source email
  - `confidence`: 0.7
- Shows a brief toast confirmation "Saved to ideas"
- Button changes to a checkmark after save

### Files
- **EDIT**: `src/components/email/ContentDigestSection.tsx` — add save button per key point row (~30 lines added)

## Feature C: Save from Email Summary Items

**Enhancement to `EmailSummaryCard.tsx`**

Each summary item row (the narrative bullet points under themed sections) gets a subtle capture affordance.

### Behavior
- On hover, a small "+" icon appears to the right of each summary item
- Clicking it opens a **mini capture popover** (lightweight, inline):
  ```
  ┌──────────────────────────┐
  │ Save as: ● Task  ○ Idea │
  │ [Editable title field]   │
  │        [Save →]          │
  └──────────────────────────┘
  ```
- Title pre-fills from the summary item text
- Type defaults to "Task" if `action_needed` flag is true, "Idea" otherwise
- Clicking "Save" creates a project_item via the project-items API
- If the summary item has `email_ids`, links the project_item via `source_email_id`

### Files
- **NEW**: `src/components/home/SummaryItemCapture.tsx` — mini capture popover (~120 lines)
- **EDIT**: `src/components/home/EmailSummaryCard.tsx` — add capture button to `SummaryItemRow` component

## Feature D: Task → Email Quick Action

**Enhancement to task/project item cards**

When a project_item or action has a `source_email_id`, show a quick "Reply" or "Open Email" action.

### Behavior
- On TriageActionCard and board items: if `source_email_id` exists, show a mail icon button
- Clicking it either:
  - **Opens the source email** in EmailDetailModal (for context review)
  - **Or** opens Gmail compose pre-filled with reply (same pattern as QuickActionButtons.handleReply)
- For items with a `contact_id`, show a "Send Email" action that opens ComposeEmail pre-filled with the contact's email

### Files
- **EDIT**: `src/components/projects/TriageActionCard.tsx` — add email icon button (~15 lines)
- **EDIT**: `src/components/projects/TriageIdeaCard.tsx` — add email icon button for ideas with source email (~15 lines)

## Implementation Order

1. **Feature A** (SmartCaptureBar) — Highest impact, most visible
2. **Feature E** (Upgrade existing save buttons) — Quick win, improves existing UX
3. **Feature B** (Key point saves) — Small addition, consistent with A
4. **Feature C** (Summary item capture) — Home page capture
5. **Feature D** (Task → Email) — Bidirectional bridge

## New Dependencies

None — all features reuse existing components (QuickAcceptPopover, CreateItemDialog, useProjectItems, useProjects) and APIs (/api/ideas, /api/project-items).

## Estimated Scope

- ~3-4 new files
- ~5-6 edited files
- No database migrations needed
- No new API routes needed
- All features use existing hooks and APIs
